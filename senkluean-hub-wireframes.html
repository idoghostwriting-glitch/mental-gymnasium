03Interactive architecture — data flow map DATA → render → DOM → delegate → Store → localStorage

The loop: a control's data-action is caught by the single delegated listener → it writes to Store (which persists to localStorage) → the relevant render*() re-runs → the search index is rebuilt via stampSearch(). On next visit, init() replays saved state so the hub reopens exactly where it was left.
